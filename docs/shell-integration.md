# Shell Integration

## The core problem

A Go process is a child of the shell that launched it. A child process cannot change the working directory of its parent — `os.Chdir()` only affects the Go process itself and is discarded when it exits.

## The solution: eval delegation

rift splits responsibility between the binary and a shell wrapper function:

1. The **shell wrapper** (named `rift`) intercepts the command and runs the binary, capturing its stdout.
2. The **binary** (`rift`) prints a `cd <path>` string to stdout — and nothing else.
3. The wrapper feeds that string back to the shell via `eval`, which executes `cd` in the current session.

```
user types: rift api
      │
      ▼
shell function `rift`
      │  calls binary, captures stdout
      ▼
rift binary
      │  prints to stdout: cd "/Users/alice/work/api"
      ▼
shell function evals captured output
      │
      ▼
shell session changes directory
```

All other output (errors, confirmations, help text) goes to **stderr**, which the terminal displays directly without `eval` ever seeing it.

## Stdout vs stderr rule

| Output type | Stream | Why |
|---|---|---|
| `cd <path>` (navigation) | stdout | shell wrapper evals this |
| Error messages | stderr | displayed directly, never evaled |
| Confirmation messages | stderr | displayed directly, never evaled |
| Help / usage text | stderr | displayed directly, never evaled |

If anything other than a valid shell command reaches stdout, the shell will try to execute it and produce confusing errors.

## Shell wrapper snippets by shell

The wrapper function is always named `rift`, shadowing the binary of the same name. Each shell needs a mechanism to call the actual binary from inside the function without recursing infinitely.

### bash / zsh (macOS, Linux, WSL, Git Bash, Cygwin, MSYS2)

```bash
rift() { eval "$(command rift "$@")"; }
```

`command rift` bypasses shell functions and aliases, calling the binary directly (POSIX standard).

Config files:
- zsh: `~/.zshrc`
- bash on macOS: `~/.bash_profile`
- bash on Linux / Git Bash / Cygwin / MSYS2: `~/.bashrc`

### fish (macOS, Linux, WSL)

```fish
function rift
    eval (command rift $argv)
end
```

fish's `command` builtin works the same way — it skips functions and calls the binary.

Config file: `~/.config/fish/config.fish`

### ksh / mksh (Linux, macOS)

```sh
rift() { eval "$(command rift "$@")"; }
```

ksh supports the same POSIX function syntax and `command` builtin as bash.

Config file: `~/.kshrc`

### PowerShell 7+ / pwsh (Windows, macOS, Linux)

**Windows:**
```powershell
function rift { Invoke-Expression (rift.exe $args) }
```

On Windows the binary is `rift.exe`. Calling `rift.exe` explicitly inside a function named `rift` avoids recursion — the `.exe` extension bypasses PowerShell's function lookup.

Config file: `~/Documents/PowerShell/Microsoft.PowerShell_profile.ps1`

**macOS / Linux:**
```powershell
function rift { Invoke-Expression (& (Get-Command -CommandType Application rift) $args) }
```

`Get-Command -CommandType Application` looks up only executables, skipping the function.

Config file: `~/.config/powershell/Microsoft.PowerShell_profile.ps1`

### Windows PowerShell 5.x (built-in on Windows)

```powershell
function rift { Invoke-Expression (rift.exe $args) }
```

Same snippet as pwsh on Windows.

Config file: `~/Documents/WindowsPowerShell/Microsoft.PowerShell_profile.ps1`

## CMD limitation

Windows Command Prompt does not have shell functions. There is no equivalent to `eval` in CMD that can change the parent session's directory. Users on CMD should switch to PowerShell, Git Bash, or WSL.

## How `rift awaken` works

`rift awaken` detects the current shell, resolves the correct config file path, checks whether the snippet is already present (via a marker comment), and appends the function if not. It then prints the `source` or `. $PROFILE` command to reload.

Detection logic in order:
1. `$SHELL` env var — covers macOS, Linux, WSL, Git Bash, Cygwin, MSYS2
2. `$PSModulePath` — set by every PowerShell session on all platforms
3. `$COMSPEC` without `$PSModulePath` — Windows CMD
4. OS fallback — Linux/macOS default to bash; Windows unknown terminal returns an error
